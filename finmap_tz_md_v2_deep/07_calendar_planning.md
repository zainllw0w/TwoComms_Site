# 07. Календарь и планирование платежей

## 1. Назначение

`Календар` показывает денежный прогноз по дням. Это финансовая сетка, где каждый день отображает остаток денег, доходы, расходы и платежи, которые должны пройти.

## 2. Общий экран

Структура:
1. Заголовок месяца, например `травень 2026`.
2. Навигация по месяцам.
3. Кнопка `Фільтр`.
4. Ссылка `Подивіться як користуватися розділом календар`.
5. Сетка дней недели: `пн`, `вт`, `ср`, `чт`, `пт`, `сб`, `нд`.
6. Ячейки дней с остатками/платежами.
7. Панель выбранного дня под календарем.

## 3. Ячейка дня

Каждая ячейка показывает:
- номер дня;
- остаток/прогноз остатка;
- если есть платежи — положительную сумму доходов и отрицательную сумму расходов;
- конечный остаток.

Пример логики: в выбранный день может быть `+12 578 568 ₴`, `-9 493 297 ₴` и итог на конец дня.

## 4. Выбранный день

При клике по дню показывать:
- `Залишок на рахунку`;
- `Платежі за день`;
- `Залишок на кінець дня`;
- `Курс до валюти компанії`;
- `Залишок в валюті компанії`;
- таблицу платежей за день.

Таблица платежей дня:
- дата/время;
- тип;
- сумма;
- счет;
- категория;
- контрагент;
- проект;
- комментарий;
- статус.

## 5. Расчет календаря

Для каждого дня:
`start_balance_day = end_balance_previous_day`

`actual_income_day = sum(actual income where date_actual = day)`

`actual_expense_day = sum(actual expense where date_actual = day)`

`planned_income_day = sum(planned income where planned_date = day)`

`planned_expense_day = sum(planned expense where planned_date = day)`

`end_balance_day = start_balance_day + actual_income_day - actual_expense_day + planned_income_day - planned_expense_day`

Переводы внутри компании не меняют общий итог всех счетов, но меняют остатки конкретных счетов.

## 6. Прошлые, текущие и будущие даты

- Прошлые даты: фактический остаток на конец дня.
- Сегодня: фактические операции + неоплаченные планы на сегодня.
- Будущие даты: прогноз с учетом плановых и повторяющихся платежей.

## 7. Фильтр календаря

Фильтровать по:
- счетам;
- проектам;
- категориям;
- контрагентам;
- тегам;
- типам платежей;
- плановые/фактические.

Фильтр меняет отображение и расчет текущего вида, но не меняет данные.

## 8. Создание платежа из календаря

Из выбранного дня пользователь должен иметь возможность:
- добавить доход;
- добавить расход;
- добавить перевод;
- создать плановый платеж.

Дата выбранного дня подставляется автоматически.

## 9. Плановые и повторяющиеся платежи

Плановые платежи видны в календаре до фактического проведения. При подтверждении становятся actual. Если просрочены — подсветить и предложить перенести или отметить оплаченным.

Повторяющиеся платежи генерируются по правилам recurrence. Не создавать бесконечные записи; генерировать occurrences на лету или на ближайшие N месяцев.

## 10. Acceptance criteria

Готово, если календарь отображает месячную сетку, остаток/прогноз по дням, детализацию выбранного дня, плановые платежи, повторения, фильтры и создание платежей из дня.

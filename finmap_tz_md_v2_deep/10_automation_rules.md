# 10. Автоправила

## 1. Назначение

Автоправила автоматически заполняют поля платежей: категории, проекты, контрагентов, теги, комментарии. Особенно важно для импортированных банковских операций.

## 2. Основной экран

Если правил нет:
- показать блок “создайте правила один раз”;
- объяснить экономию времени;
- CTA `Створити своє перше правило`.

Если правила есть:
- список правил;
- включено/выключено;
- тип доход/расход;
- условия;
- действия;
- приоритет;
- количество применений;
- редактирование;
- удаление.

## 3. Модалка “Нове правило”

Поля:
1. `Ім'я`;
2. переключатель `Вимкнути правило`;
3. `Правило буде застосовуватися для:` `Дохід` / `Витрата`;
4. блок `Умова`;
5. условие: поле, оператор, значение;
6. `+ Додати ще одну умову`;
7. блок действий;
8. действие, поле назначения, значение;
9. `+ Додати ще одну дію`;
10. `Додати правило`;
11. `Відміна`.

## 4. Условия

Условие = field + operator + value.

Поля:
- комментарий/назначение платежа;
- сумма;
- счет;
- контрагент;
- категория;
- проект;
- тег;
- дата;
- источник;
- валюта;
- тип операции;
- IBAN/банк;
- external description.

Операторы:
- содержит, не содержит, равно, не равно;
- начинается с, заканчивается;
- больше, меньше, между;
- до, после, в диапазоне;
- входит в, не входит в.

Для MVP достаточно логики AND между условиями.

## 5. Действия

- установить категорию;
- установить проект;
- установить контрагента;
- добавить тег;
- установить счет;
- изменить комментарий;
- пометить как плановый/фактический;
- исключить из отчета как техническую операцию;
- отправить на подтверждение.

Для каждого действия нужна настройка:
- перезаписывать существующее значение;
- заполнять только если пусто.

## 6. Приоритеты и применение

Если подходит несколько правил, применять по `priority`. Правила применяются:
- при ручном создании как подсказка;
- при импорте выписки;
- при банковской синхронизации;
- массово к выбранным платежам.

## 7. Применить к существующим

Дать опцию применить правило к существующим платежам за период. Перед применением показать preview:
- какие платежи изменятся;
- старые значения;
- новые значения;
- подтвердить/отменить.

## 8. История и откат

Логировать каждое применение: rule_id, transaction_id, before, after, user/auto, time. Для массового применения нужен откат.

## 9. Acceptance criteria

Готово, если можно создать правило с именем, типом, условиями и действиями; включить/выключить; применить к импортированным и существующим платежам; посмотреть историю применения.

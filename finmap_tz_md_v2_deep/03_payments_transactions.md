# 03. Раздел “Платежі” — журнал операций

## 1. Назначение

`Платежі` — главный рабочий экран финансового учета. Здесь отображаются доходы, расходы, переводы, плановые платежи, операции из банковских интеграций и импортированных выписок.

Пользователь должен быстро:
- найти платеж;
- понять, по какому счету он прошел;
- увидеть остаток после операции;
- проверить контрагента, категорию, проект и комментарий;
- открыть платеж и исправить данные;
- создать новый платеж через верхние кнопки.

## 2. Расположение элементов

В рабочей области справа от левой панели:
1. верхнее меню разделов;
2. панель фильтров;
3. таблица операций;
4. группа `Планові платежі • N`;
5. вертикальный скролл;
6. редактирование по клику на строку.

## 3. Фильтры

### 3.1 Период
Первый фильтр, пример `За весь час`. Варианты: за весь час, сегодня, вчера, текущая неделя, текущий месяц, прошлый месяц, квартал, год, произвольный диапазон.

### 3.2 Поиск
Placeholder: `Пошук по рахунках, клієнтах, коментарях`.

Искать по счету, контрагенту, комментарию, категории, проекту, тегам, сумме, номеру инвойса, external_id.

### 3.3 Сумма
Блок `Сума` с полями `Від` и `До`. Можно указать только минимум или максимум. По умолчанию фильтровать по абсолютной сумме.

### 3.4 Расширенный фильтр
Кнопка `Фільтр` открывает модалку с фильтрами по счетам, категориям, проектам, контрагентам, тегам, типам и статусам.

### 3.5 Обновление, экспорт, импорт
Иконка обновления перечитывает данные и может запускать безопасную синхронизацию. Иконки скачивания/загрузки: экспорт CSV/XLSX и импорт выписки.

## 4. Таблица операций

Колонки:
1. чекбокс;
2. `Дата` — дата и время;
3. `Сума` — доходы положительные, расходы отрицательные;
4. `Рахунок/залишок` — счет и остаток после операции;
5. `Контрагент`;
6. `Категорія`;
7. `Проект`;
8. `Коментар`.

Дополнительно можно хранить: источник, статус, пользователь, дата создания, интеграция.

## 5. Строка операции

Пример дохода:
- дата `24 вер 2025`, время `19:14`;
- сумма `+53 362 ₴`;
- счет `Приват Карта`, остаток `485 988 ₴`;
- контрагент `Вказати`;
- категория `Інші доходи`;
- проект `PromUA`;
- комментарий `Поповнення від Єгор Герасименко`.

Пример расхода:
- сумма `-52 191 ₴`;
- счет `Приват Карта`;
- категория `Товари з Європи`;
- проект `PromUA`;
- комментарий `Оплата закупівлі партії товару C-2`.

## 6. Группировка плановых платежей

Строка-группа `Планові платежі • 2545`:
- может сворачиваться/разворачиваться;
- показывает количество плановых платежей;
- отделяет плановые операции от фактических;
- плановые операции не меняют фактический баланс до подтверждения.

## 7. Клик по строке

Клик открывает форму редактирования:
- доход — `Дохід`;
- расход — `Витрата`;
- перевод — `Переказ`.

В форме можно менять счет, сумму, дату, категорию, контрагента, проект, теги, комментарий и файл.

## 8. Массовые действия

Чекбоксы позволяют выбрать несколько строк. Нужны массовые действия:
- изменить категорию;
- изменить проект;
- изменить тег;
- изменить контрагента;
- удалить;
- экспортировать выбранные;
- применить автоправило;
- отметить плановые как фактические.

## 9. Источники платежей

Платеж может быть создан вручную, из банковской интеграции, из импортированной выписки, из инвойса, из автоправила, как повторение, через AI advisor.

## 10. Поведение после изменений

После создания/редактирования/удаления:
1. пересчитать баланс счетов;
2. обновить левую панель;
3. обновить платежный журнал;
4. обновить календарь;
5. обновить отчеты;
6. записать действие в `AuditLog`.

## 11. Состояния

- Нет платежей: показать CTA добавить доход/расход/перевод.
- Нет результатов по фильтру: показать кнопку сброса.
- Ошибка загрузки/синхронизации: показать сообщение и кнопку повторить.

## 12. Acceptance criteria

Раздел готов, если отображает операции с колонками из скриншота, поддерживает период/поиск/сумму/расширенный фильтр, строки открываются на редактирование, плановые платежи отделены, а изменения пересчитывают счета и отчеты.

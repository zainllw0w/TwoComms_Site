# 08. Рахунки-фактури / инвойсы

## 1. Назначение

Раздел `Рахунки-фактури` нужен для создания счетов клиентам, хранения реквизитов, контроля оплат и связи выставленных счетов с платежами.

## 2. Основной экран

Верхняя часть:
1. Вкладка `Рахунки-фактури`.
2. Вкладка `ПДВ`.
3. Кнопка `+ Новий рахунок`.
4. Фильтр периода, например `За поточний рік`.
5. Кнопка `Фільтр`.
6. Обновление.
7. Ссылка обучения.

Если счетов нет, показать промо-блок с объяснением и CTA.

## 3. Список счетов

Колонки:
- номер счета;
- дата создания;
- дата оплаты/дедлайн;
- получатель/плательщик;
- сумма;
- валюта;
- статус;
- оплачено;
- остаток к оплате;
- проект;
- действия.

Статусы: draft, issued, sent, partially_paid, paid, overdue, cancelled.

## 4. Модалка “Новий рахунок”

### 4.1 Верхняя зона
Слева:
- зона загрузки логотипа;
- `Завантажте сюди логотип компанії`;
- `Виберіть файл`.

Справа:
- `Номер рахунку`;
- `Дата створення рахунка`;
- `Дата оплати рахунка`;
- `Валюта рахунку`.

### 4.2 Данные поставщика
Блок `Дані постачальника`:
- `Назва компанії`;
- `ЄДРПОУ`;
- `IBAN`;
- `Країна`;
- `Адреса`.

### 4.3 Данные получателя/плательщика
Блок `Дані отримувача (платника)`:
- `Назва компанії`;
- `ЄДРПОУ`;
- `IBAN`;
- `Країна`;
- `Адреса`.

Выбор контрагента должен подставлять реквизиты.

### 4.4 ПДВ/НДС
Чекбокс `Враховувати ПДВ`. Если включен, у позиций есть ставка налога и итоговая налоговая сумма.

### 4.5 Позиции
Строка позиции:
- `Товар/Назва Послуги`;
- `Кількість`;
- `Ціна`;
- `Сума`;
- удаление строки.

Действие `+ Додати позицію`.

Формула: `item_amount = quantity * unit_price`.

### 4.6 Скидка, доставка, итог
Действия:
- `+ Додати знижку`;
- `+ Врахувати доставку`.

Итог:
`total = subtotal - discount + delivery + tax`.

Показывать `Загальна сума`.

### 4.7 Дополнительная информация
Блок `Додаткова інформація`: поле `Нотатки`, условия оплаты, комментарий.

## 5. Связь счета с платежами

Счет может быть:
- не оплачен;
- частично оплачен;
- оплачен.

Оплата счета создает/связывает платеж с `invoice_id`. Если оплат несколько, статус пересчитывается по сумме оплат.

## 6. ПДВ вкладка

Показывать:
- счет;
- дата;
- контрагент;
- сумма без ПДВ;
- ПДВ;
- сумма с ПДВ;
- статус оплаты;
- период и экспорт.

## 7. Экспорт и отправка

Нужно:
- скачать PDF;
- скачать XLSX;
- отправить по email;
- скопировать ссылку;
- отметить как отправленный.

## 8. Acceptance criteria

Готово, если есть вкладки счетов и ПДВ, создание счета с реквизитами и позициями, ПДВ, скидка, доставка, корректный итог, связь с оплатами, статусы и PDF.

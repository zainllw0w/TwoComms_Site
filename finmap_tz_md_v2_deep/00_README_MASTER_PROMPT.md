# FinMap-like сервис управления финансами — комплект ТЗ для AI-агента

Этот набор `.md`-файлов описывает, как реализовать веб-сервис управления финансами компании по скриншотам FinMap. Документы написаны не как дизайн-гайд, а как функциональное ТЗ: где что находится, какие сущности нужны, какие действия выполняются, как должны считаться суммы, какие модальные окна и отчеты должны существовать.

## Как использовать эти файлы с AI-разработчиком

Передайте агенту всю папку с `.md` файлами и дайте такой промпт:

> Реализуй веб-приложение для управления финансами компании по ТЗ из папки finmap_tz_md_v2_deep. Не копируй визуальный стиль 1-в-1, но повтори структуру продукта, навигацию, разделы, сущности, модальные окна, таблицы, фильтры, расчеты, отчеты, интеграции и поведение. Основной источник правды — файлы `01`–`12`. Если между файлами есть противоречия, приоритет такой: `01_overview_navigation_layout.md`, затем `02_data_model_and_calculations.md`, затем конкретный файл раздела.

## Состав ТЗ

1. `01_overview_navigation_layout.md` — общая структура приложения, верхняя панель, левая финансовая панель, вкладки, роли экранов.
2. `02_data_model_and_calculations.md` — сущности, связи, суммы, балансы, валюты, плановые платежи, переносы, повторения.
3. `03_payments_transactions.md` — журнал платежей, таблица, фильтры, поиск, строки операций.
4. `04_transaction_modals_income_expense_transfer.md` — модалки дохода, расхода, перевода, редактирования, повторения, вложения.
5. `05_accounts_integrations.md` — счета, остатки, добавление счетов, интеграции банков/сервисов, импорт.
6. `06_analytics_reports.md` — аналитика и отчеты.
7. `07_calendar_planning.md` — календарь, плановые платежи, прогноз остатков по дням.
8. `08_invoices.md` — рахунки-фактури/инвойсы, ПДВ, создание счета, товары/услуги, статусы.
9. `09_ai_advisor.md` — AI-радник.
10. `10_automation_rules.md` — автоправила.
11. `11_users_company_settings_effectiveness.md` — компании, пользователи, доступы, эффективность.
12. `12_common_components_filters_tables.md` — общие фильтры, таблицы, дропдауны, модалки.
13. `13_screenshot_manifest.md` — индекс 61 скриншота.
14. `14_IMPLEMENTATION_CHECKLIST.md` — чек-лист реализации.

## Важные принципы реализации

- Интерфейс должен быть модульным: каждый раздел работает отдельно, но использует общие сущности.
- Все денежные операции должны проходить через единую модель `Transaction`.
- Счета и балансы — центр системы. Любое создание/редактирование/удаление платежа должно пересчитывать остатки.
- Плановые платежи не равны фактическим: они участвуют в прогнозе, календаре и “балансе с учетом будущих платежей”, но не должны менять фактический баланс до подтверждения/оплаты.
- Раздел аналитики должен строиться из тех же данных, что и платежи: не хранить отчеты как отдельные ручные таблицы, а рассчитывать их из транзакций, счетов, проектов, категорий, контрагентов.
- Дизайн можно адаптировать под уже существующую систему, но расположение зон, навигационная логика, сценарии и состав функций должны сохраняться.

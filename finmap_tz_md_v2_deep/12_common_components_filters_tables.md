# 12. Общие компоненты: фильтры, таблицы, дропдауны, модалки

## 1. Цель

Реализовать единые компоненты для всех разделов: фильтры, мультиселекты, таблицы, модалки, date picker, file uploader, графики, empty/loading/error states.

## 2. Общий фильтр

Модалка `Фільтр` содержит:
- период/дата;
- счет;
- категория;
- проект;
- контрагент;
- тег;
- тип операции;
- статус.

Кнопки:
- `Пошук`;
- `Скинути все`.

Фильтр должен сохраняться в URL/query или состоянии страницы и иметь быстрый сброс.

## 3. Мультиселекты

### 3.1 Теги
`Всі теги`, чекбоксы, `Повна оплата`, `Аванс`, `Без тегів`, кнопка `Обрати`.

### 3.2 Счета
`Всі рахунки`, список счетов, чекбоксы, кнопка `Обрати`.

### 3.3 Проекты
`Всі проекти`, проекты, `Без проектів`, кнопка `Обрати`.

### 3.4 Контрагенты
`Всі контрагенти`, группы `Постачальники`, `Кредитори`, `Мої Контрагенти`, `Власники`, раскрытие групп, чекбоксы, кнопка `Обрати`.

## 4. Период отчета

Варианты:
- `На поточний момент`;
- `Вчора (на кінець дня)`;
- `На кінець минулого місяця`;
- `Вибрати дату (на кінець дня)`;
- за весь час;
- текущий месяц;
- прошлый месяц;
- год;
- произвольный диапазон.

## 5. Таблица

Компонент таблицы:
- заголовки;
- сортировка;
- скролл;
- sticky header;
- чекбоксы;
- empty state;
- loading;
- row click;
- контекстные действия;
- экспорт.

Колонки задаются конфигурацией.

## 6. Модалки и боковые панели

Типы:
- центральная модалка;
- правая боковая панель;
- полноэкранная модалка.

Правила:
- затемнение фона;
- крестик закрытия;
- Esc закрывает;
- предупреждение при несохраненных изменениях;
- primary/secondary actions;
- валидация;
- loading при сохранении.

## 7. Date/time picker

Поддержать дату, время, “сегодня”, календарный выбор, timezone. Формат в формах: `Сьогодні 29.05.2026, 16:30`.

## 8. File uploader

Используется в платежах и инвойсах:
- drag&drop;
- выбрать файл;
- максимум 10 МБ;
- список файлов;
- удалить файл;
- превью изображения/PDF;
- безопасное хранение.

## 9. Графики

Типы:
- столбчатые;
- стековые;
- круговые;
- tooltip;
- легенда;
- экспорт.

Графики должны реагировать на фильтры.

## 10. Форматы чисел и валют

Правила:
- разделитель тысяч пробелом;
- дробная часть до 2 знаков;
- валюта рядом;
- доходы со знаком плюс;
- расходы со знаком минус.

Примеры: `₴ 3 663 056.73`, `+53 362 ₴`, `-52 191 ₴`, `€ 5 950`, `$ 1 650`.

## 11. Создание сущности из дропдауна

Для проектов, контрагентов, категорий и тегов:
- показать `+ Створити ...`;
- открыть маленькую форму;
- после сохранения выбрать новое значение.

## 12. Ошибки

Показывать ошибки рядом с полем и toast:
- обязательное поле;
- неправильная сумма;
- ошибка сети;
- недостаточно прав;
- интеграция недоступна;
- файл слишком большой;
- дубль операции.

## 13. Acceptance criteria

Готово, если фильтры работают одинаково во всех разделах, мультиселекты поддерживают чекбоксы и группы, таблицы имеют сортировку/loading, модалки валидируют данные, файлы загружаются, графики реагируют на фильтры.

# Gemini Art Dept — WOW Pass (DTF Redesign / Control Deck × Lab Proof)

Ты — Art/Asset генератор для **dtf.twocomms.shop**. Код и UI готовы. Твоя задача — заменить плейсхолдеры финальными ассетами и поднять «вау» до лабораторного уровня. **Файлы и имена строго фиксированы.**

## 0) Невозможные ошибки (жёстко)
- **Не менять имена файлов и пути.**
- **Не добавлять текст** в изображения, если это явно не требуется.
- **Одинаковый кадр** для before/after (если есть пары).
- **Без AI‑артефактов**, смазов, полос, «мыла».
- sRGB, нормальная компрессия, аккуратные края.

## 1) Визуальный язык (повтор ключевых правил)
- **Стиль:** industrial, lab‑grade, “Control Deck × Lab Proof”.
- **Свет:** нейтральный, чистый, без warm‑tone.
- **Палитра:** carbon black + subtle metal + molten orange акцент.
- **Композиция:** минимально, технично, без визуального мусора.
- **Избегать:** дешёвого CGI, «кибер‑неона», ярких градиентов, лишних отражений.

## 2) Критические ассеты (ЗАМЕНИТЬ плейсхолдеры)

### A) Gallery macro set (реальные фото/рендеры)
**Нужно:** 8–12 реальных макро/процесс/финал кадров, один стиль освещения.
**Файлы:**
- `twocomms/dtf/static/dtf/assets/gallery-macro-01.jpg`
- `.../gallery-macro-02.jpg`
- ... до `gallery-macro-12.jpg`
**Примечания:**
- Сетка на главной и в галерее использует эти файлы — это основной «вау».
- Никаких текстов, логотипов, брендирования в кадрах.
- Макро‑текстура (край, подложка, точность линий, микро‑шаг печати).

### B) Lab Proof set (QC / Lab)
**Нужно:** 4–6 фото, «лабораторные доказательства».
**Файлы:**
- `twocomms/dtf/static/dtf/assets/lab-proof-01.jpg`
- `.../lab-proof-02.jpg`
- ... до `lab-proof-06.jpg`
**Примечания:**
- Темы: registration, white underbase, wash test, edge/tactile, QC check.
- Строгий, чистый, лабораторный стиль.

### C) Template source files (НЕ SVG‑заглушки)
**Нужно:** реальные исходники, а не переименованный SVG.
- `twocomms/dtf/static/dtf/assets/template-60cm.fig`
- `twocomms/dtf/static/dtf/assets/template-60cm.ai`
- `twocomms/dtf/static/dtf/assets/template-60cm.pdf`
**Требования к шаблону:**
- Чёткие guide‑линии, safe‑margins, bleed, служебные подписи (если допустимо внутри файла).
- Без декоративных иллюстраций.
- PDF — корректный экспорт 1:1, без артефактов.

## 3) Можно улучшить (если качество будет заметно выше)
Если можешь сделать **существенно лучше**, можно улучшить существующие ассеты (но только при явном выигрыше):
- `lens-macro-1.jpg`, `gallery-lens-macro-1.jpg`
- `compare-case-1/2-before/after.jpg`
- `gallery-compare-1/2/3-before/after.jpg`

**Если улучшений нет — не трогай.**

## 4) Технический контроль
Перед отдачей проверь:
- Все JPG резкие (без banding), без артефактов.
- Все размеры достаточно большие (2x от отображения).
- Цвета не уходят в «тёплый» тон.
- Пары «до/после» совпадают по кадру.

## 5) Итог
Отдай готовые файлы **строго** по путям. Никаких переименований.
Если какой‑то формат невозможен, сообщи и предложи максимально близкий аналог.

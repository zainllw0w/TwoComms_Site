# 📋 Пошаговая инструкция: Настройка CRON через cPanel

## 🎯 Цель

Настроить автоматическую проверку статусов Nova Poshta каждые 5 минут через интерфейс cPanel.

---

## 📍 ШАГ 1: Вход в cPanel

1. Откройте браузер и войдите в **cPanel** вашего хостинга
2. Обычно это: `https://ваш-домен.com/cpanel` или `https://195.191.24.169:2083`

---

## 🔍 ШАГ 2: Найти раздел Cron Jobs

### Вариант A: Через поиск (самый простой)

1. В верхней части cPanel найдите **поле поиска** (Search)
2. Введите: **`cron`** или **`запланированные задачи`**
3. Нажмите на найденный результат

### Вариант B: Через меню

1. Найдите раздел **"Дополнительно"** или **"Advanced"**
2. Внутри найдите **"Cron Jobs"** или **"Запланированные задачи"**

### Вариант C: Прямая ссылка

Если у вас есть доступ к URL:
```
https://ваш-домен/cpanel/cron/index.html
```

---

## ✏️ ШАГ 3: Добавить или изменить задачу

### Вариант A: Если задача УЖЕ существует

1. Прокрутите страницу вниз до раздела **"Текущие задачи Cron"** или **"Current Cron Jobs"**
2. Найдите строку с текстом `update_tracking_statuses`
3. Нажмите кнопку **"Изменить"** (Edit) рядом с этой задачей
4. Перейдите к **ШАГ 4**

### Вариант B: Если задачи НЕТ (создать новую)

1. Прокрутите страницу вниз до раздела **"Добавить новую задачу Cron"** или **"Add New Cron Job"**
2. Выберите переключатель: **"Стандарт (Advanced)"** или **"Standard (Advanced)"**
   - ⚠️ **НЕ выбирайте "Простой" (Basic)** - там нет нужных полей
3. Перейдите к **ШАГ 4**

---

## ⚙️ ШАГ 4: Заполнить форму

### Поля времени (Расписание):

Заполните следующие поля:

| Поле | Значение | Объяснение |
|------|----------|------------|
| **Минута** | `*/5` | Каждые 5 минут |
| **Час** | `*` | Каждый час |
| **День** | `*` | Каждый день месяца |
| **Месяц** | `*` | Каждый месяц |
| **Неделя** | `*` | Каждый день недели |

**Итого строка расписания:** `*/5 * * * *`

### Поле команды:

В поле **"Команда"** (Command) вставьте следующую команду:

```bash
cd /home/qlknpodo/TWC/TwoComms_Site/twocomms && /home/qlknpodo/virtualenv/TWC/TwoComms_Site/twocomms/3.14/bin/python manage.py update_tracking_statuses >> /home/qlknpodo/TWC/TwoComms_Site/twocomms/logs/nova_poshta_cron.log 2>&1
```

**Или более короткий вариант** (если первый не работает):

```bash
cd /home/qlknpodo/TWC/TwoComms_Site/twocomms && /home/qlknpodo/virtualenv/TWC/TwoComms_Site/twocomms/3.14/bin/python manage.py update_tracking_statuses >> /tmp/nova_poshta.log 2>&1
```

---

## 💾 ШАГ 5: Сохранить

1. Прокрутите страницу вниз
2. Нажмите кнопку **"Добавить новую задачу Cron"** или **"Add New Cron Job"** 
   - Или **"Сохранить изменения"** / **"Save Changes"** (если редактировали существующую)
3. Должно появиться сообщение об успехе: **"Cron job added successfully"** или подобное

---

## 🔄 Что делает эта задача

Каждые **5 минут** автоматически:

1. ✅ Проверяет все заказы с ТТН через API Новой Почты
2. ✅ Обновляет статусы посылок (`shipment_status`)
3. ✅ Когда посылка получена → автоматически:
   - Меняет статус заказа на **"Отримано"** (`done`)
   - Меняет статус оплаты на **"Оплачено"** (`paid`)
   - Отправляет Purchase событие в **Meta Pixel** (Facebook)
   - Отправляет уведомления в **Telegram** админу и клиенту

**Всё это происходит АВТОМАТИЧЕСКИ, без вашего участия!**

## 🆘 Решение проблем

### Проблема 1: "Файл не найден" или "Permission denied"

**Решение:**
1. Убедитесь что директория для логов существует:
   ```bash
   mkdir -p /home/qlknpodo/TWC/TwoComms_Site/twocomms/logs
   chmod 755 /home/qlknpodo/TWC/TwoComms_Site/twocomms/logs
   ```

2. Или используйте временный путь `/tmp/nova_poshta.log`

### Проблема 2: Задача не запускается

**Проверьте:**
1. Правильность пути к Python
2. Правильность пути к проекту
3. Логи на наличие ошибок

### Проблема 3: "Command not found"

**Решение:**
Используйте полный путь к Python (как в команде выше)

## 📝 Расшифровка расписания

### `*/5 * * * *` означает:

- **`*/5`** - каждые 5 минут (00, 05, 10, 15, 20, 25, 30, 35, 40, 45, 50, 55)
- **`*`** - каждый час
- **`*`** - каждый день месяца
- **`*`** - каждый месяц
- **`*`** - каждый день недели

**Итого:** Команда запускается **каждые 5 минут, 24 часа в сутки, 7 дней в неделю**.

### Другие примеры расписания:

| Расписание | Когда запускается |
|------------|-------------------|
| `*/5 * * * *` | Каждые 5 минут |
| `*/15 * * * *` | Каждые 15 минут |
| `0 * * * *` | Каждый час (в :00) |
| `0 0 * * *` | Каждый день в полночь |
| `0 0 * * 0` | Каждое воскресенье в полночь |

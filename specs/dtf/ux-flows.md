# DTF Subdomain — UX Flows

## Flow 1: Landing → Order (Ready)
1) Пользователь заходит на `dtf.twocomms.shop`.
2) Видит hero-оффер, CTA «Рассчитать / Заказать».
3) Переходит на /order (таб «Есть готовый ганг-лист»).
4) Загружает файл, вводит длину, копии, контактные данные, НП.
5) Видит расчет метража/цены, отправляет форму.
6) Получает страницу «Спасибо»: номер заказа, статус «Проверка макета», ссылка на /status.

## Flow 2: Landing → Order (Help)
1) Пользователь нажимает «Нужна помощь с макетом».
2) Переходит /order?tab=help.
3) Загружает несколько файлов или указывает ссылку, описывает задачу и сроки.
4) Оставляет контакты и НП.
5) Получает страницу «Спасибо»: номер заявки, сообщение о связи менеджера.

## Flow 3: Status Tracking
1) Пользователь открывает /status.
2) Вводит номер заказа + телефон.
3) Видит статусную цепочку и подсказку «что дальше».
4) При наличии ТТН — видит номер отправления.

## Flow 4: Lead (FAB)
1) Нажимает плавающую кнопку «Менеджер».
2) В модальном окне вводит имя, контакт, канал связи, вопрос.
3) Отправляет → получает сообщение «Менеджер свяжется».

## Acceptance Criteria
- [ ] Два основных потока заказа (готовый/помощь) работают end-to-end.
- [ ] /status доступен и корректно ищет заказ по номеру и телефону.
- [ ] FAB-форма создает Lead и отправляет Telegram уведомление.

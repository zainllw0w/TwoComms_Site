# 12 — SEO + Performance + Accessibility (P1/P2)

## 1) SEO‑база (без “магии”, только фундамент)

### Метаданные
- У каждой ключевой страницы: уникальные `title` и `description` (UA/RU/EN).
- OpenGraph/Twitter карточки (title/desc/image).
- Canonical URL (особенно при языковых префиксах/параметрах).

### Индексация
- `robots.txt` и `sitemap.xml` (если в проекте это предусмотрено).
- Проверить, что языковые версии не создают дубли (hreflang, canonical стратегия).

### Structured Data (Schema.org)
Высокая уверенность:
- Organization
- WebSite (SearchAction — если есть поиск)
- BreadcrumbList (на внутренних)

Осторожно:
- FAQ schema — использовать аккуратно, без надежды на rich snippets (Google может не показывать).
- Product/Offer — только если реально есть “товар” со стабильной ценой/наличием.

---

## 2) Content/перелинковка (минимум)
- На страницах “Вимоги”, “Ціни”, “Зразок”, “QC” сделать перекрёстные ссылки:
  - “Вимоги” → “Шаблони 60 см” → “Перевірка файлу”
  - “Ціни” → “Як рахувати метраж” → “Замовити”
  - “Зразок” → “QC” → “Вимоги”

---

## 3) Performance (особенно важно для мобайла)

### Изображения
- Только оптимизированные форматы (webp/avif где возможно).
- Lazy‑loading на длинных страницах.
- Не грузить тяжёлые “качественные” фото выше fold без нужды (использовать постер + progressive load).

### Анимации
- Учитывать `prefers-reduced-motion`.
- Не делать анимации, завязанные на постоянные reflow.
- Скелетон/лоадер — только если есть реальное ожидание.

### JS/CSS
- Не тянуть большие библиотеки ради мелких эффектов.
- Компоненты “по страницам”: code‑splitting (если используется фреймворк).

---

## 4) Accessibility (WCAG‑база)

### Навигация
- Все интерактивы доступны с клавиатуры (Tab/Enter/Esc).
- Dropdown в хедере закрывается Esc, фокус не теряется.

### Контраст и текст
- Контраст текста на тёмном фоне достаточный.
- Ссылки/кнопки различимы не только цветом (hover/focus underline/outline).

### Touch targets
- На мобайле кнопки/иконки контактов легко нажимаются.

### Tooltip/accordion
- Tooltip должен открываться по focus и иметь ARIA‑описание.
- Accordion: правильные aria‑атрибуты, управление с клавиатуры.

---

## 5) Acceptance criteria
- Lighthouse/Pagespeed не деградирует после редизайна (в пределах здравого).
- Нет “дерганых” анимаций, нет лагов на мобильном.
- Можно пройти основной путь (расчёт/контакт/заявка) без мыши.

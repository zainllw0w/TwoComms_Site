# 16 — Библиотека микро‑эффектов (50 идей, внедрять выборочно) (P2)

## Важно
Эффекты добавляем **после** стабилизации P0/P1, и только те, что:
- помогают понять интерфейс,
- повышают ощущение качества,
- не тормозят слабые устройства,
- уважают `prefers-reduced-motion`.

---

## A) Хедер / навигация
1) Hover‑reveal контактов: иконка → капсула с текстом (плавно).  
2) Dropdown профиля: лёгкий scale+fade, фокус‑ловушка, закрытие Esc.  
3) Active‑link underline: “живая линия” при наведении (без резких прыжков).  
4) Микро‑вибрация (только mobile, если уместно): лёгкий haptic на tap (если поддерживается).  
5) Смена языка: мягкий crossfade текста, без скачков высоты.  

## B) Hero
6) One‑time toast статуса (24/7/до 12): плавный выезд сверху, auto‑dismiss, “x” закрыть.  
7) Rotating benefit text: плавная смена 3–4 фраз (без blink).  
8) “Proof badge” (Hot Peel): hover показывает tooltip “що це”.  
9) CTA hover: лёгкий “подъём” + подсветка края (subtle).  
10) Smooth anchor scroll: при клике CTA скролл к секции, но с учётом reduced motion.  

## C) Карточки/секции
11) Card hover “spotlight”: мягкое световое пятно под курсором (очень аккуратно).  
12) Card reveal: при скролле карточки появляются fade+slide (с лимитом, чтобы не тормозило).  
13) “Read more” accordion: открытие с плавным height transition (без дёрганий).  
14) Tooltip: появляется с маленькой задержкой, чтобы не мигал.  
15) Skeleton для секций с реальной загрузкой данных.  

## D) Кнопки/CTA
16) Loading state кнопки: spinner внутри, текст меняется на “Обробка…” (только если реально есть ожидание).  
17) Success state: короткий check‑icon после отправки формы.  
18) Disabled state с объяснением: tooltip “Заповніть телефон”.  
19) Ripple/ink — только если соответствует стилю, иначе не использовать.  
20) “Smart copy”: если CTA повторяется ниже, то меняем подпись (не меняя смысл).  

## E) Формы (зразок/заказ)
21) Inline validation: ошибка появляется только после blur/submit, не сразу.  
22) Phone mask: визуально помогает, но не мешает вставке номера.  
23) Stepper: шаг 1→2 как карточная колода.  
24) Progress indicator: “Крок 1/2” маленьким текстом.  
25) Автоподстановка города/НП (если есть) — с dropdown + keyboard navigation.  

## F) Калькулятор/цены
26) Live update цены: число меняется с мягкой анимацией (count‑up), без “казино”.  
27) Mini‑diagram: при вводе метров появляется маленькая схема “60 см × X м”.  
28) Таблица цен: highlight текущего диапазона в зависимости от введённого метража.  
29) Переключение вкладок: crossfade (без overlay rectangle).  
30) Empty state: вместо пустоты — короткая подсказка “Введіть метраж”.  

## G) Preflight / QA
31) Status chips: OK/INFO/WARN/FAIL с аккуратной иконкой.  
32) Expand details: “Що не так?” раскрывает блок “як виправити”.  
33) Confirm “Друкувати як є”: checkbox + анимация появления кнопки.  
34) “Copy tip”: кнопка копировать рекомендации для дизайнера.  
35) File upload: drag‑n‑drop с подсветкой зоны.  

## H) QC / галерея
36) Before/after slider: сравнение макро (линии/белый).  
37) Zoom on click: открыть фото в lightbox с pinch‑zoom на мобайле.  
38) Carousel: плавный свайп, но без heavy JS.  
39) “Quality checklist” reveal: раскрытие списка проверок QC.  
40) “Customer use case” cards: короткие кейсы (если есть контент).  

## I) Mobile UX
41) Bottom bar: subtle shadow, auto‑hide при скролле вниз, show при скролле вверх (опционально).  
42) Contact sheet: bottom sheet с действиями (Telegram/Phone).  
43) Sticky CTA: если форма длинная — маленькая плавающая кнопка “Надіслати”.  
44) Tap feedback: micro‑scale на нажатии (очень лёгкий).  
45) Input focus: мягкая подсветка поля + автоскролл, чтобы не закрыла клавиатура.  

## J) Сервисные
46) “Copied!” toast: когда скопировали номер/ник.  
47) “Saved” indicator: если сохраняете прогресс формы локально.  
48) 404 page: дружественная анимация + CTA “На головну / Розрахувати”.  
49) Offline/slow network: баннер “повільне з’єднання” (если реально детектируете).  
50) Reduced motion: переключатель в настройках (если нужно) или авто по системным настройкам.  

---

## Рекомендация по внедрению
Выбрать 10–15 эффектов, которые:
- напрямую помогают конверсии (контакты, формы, preflight),
- повышают доверие (QC),
- улучшают “ощущение качества” без перегруза.

# IDEAS/O3 — Пакет инструкций для CODEX (TwoComms DTF)

Эта папка содержит набор **операционных промптов/брифов** для CODEX (GPT‑Codex 5.3), чтобы он мог **внести изменения в проект dtf.twocomms.shop** без потери контекста и без повторных «разгонов» в каждом новом сеансе.

## Как работать (обязательно)

1. **CODEX открывает и читает** файлы в порядке:
   1) `01_MASTER_PROMPT_CODEX.md`  
   2) `02_BACKLOG_CHECKLIST.md`  
   3) остальные файлы по мере выполнения задач.

2. CODEX **ведёт список задач** в двух местах:
   - через MCP *todo/liner* (если доступно),  
   - и **обновляет чек‑лист** в `02_BACKLOG_CHECKLIST.md` (ставит `[x]`).

3. На каждом крупном шаге CODEX использует **Sequential Thinking MCP**:
   - сначала кратко формирует план,
   - затем реализует,
   - затем проверяет результат (UI/адаптив/локализации/перфоманс).

4. После каждого крупного блока изменений:
   - `git status` → `git diff`,
   - **git commit** (понятное сообщение),
   - **git push**,
   - деплой через **SSH** согласно текущему проектному процессу (см. `14_DEPLOYMENT_WORKFLOW.md`).

5. В конце CODEX создаёт/обновляет:
   - `CHANGELOG.md` (или файл по шаблону из `15_CHANGELOG_TEMPLATE.md`) с перечнем сделанного, ссылками на коммиты и скринами/описанием проверки.

## Главные цели проекта (суть)

- **Сохранить стиль** (тёмный/оранжевый, «технологичность»).
- **Уменьшить перегруз**: особенно в хедере и hero, но **не ломать** идею «контекстных CTA».
- **Сделать золотую середину**: новичок понимает, профи не раздражается.
- **Сильнее доверие**: контроль рисков, preflight, QC, честные обещания по срокам.
- **Сильнее конверсия**: меньше когнитивной нагрузки, быстрее путь к расчёту/заказу/контакту.

## Запреты (важно)

- **Никаких dark patterns**: фейковые счётчики, «сейчас 4 клиента считают», подмена формата «бесплатного» без раскрытия условий, искусственная срочность, фейковые прогресс‑бары.
- Любая срочность (до 12:00 — отправка сегодня) должна быть:
  - **реальной**, проверяемой,
  - и оформленной как **операционный статус**, а не как «баннер‑крик».

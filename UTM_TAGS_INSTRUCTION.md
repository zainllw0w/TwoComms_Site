# 📊 Инструкция по заполнению UTM-тегов для Диспетчера статистики

## 🎯 Введение

UTM-теги (Urchin Tracking Module) используются для отслеживания источников трафика и эффективности рекламных кампаний. Правильное заполнение UTM-тегов критически важно для корректной работы статистики в Диспетчере.

---

## ❌ Анализ вашего текущего формата

### Ваш текущий формат:
```
utm_source={{campaign.id}}&utm_medium=Instagram&utm_campaign={{campaign.name}}&utm_content={{ad.name}}&Creo={{placement}}
```

### 🔴 Обнаруженные проблемы:

#### 1. **utm_source={{campaign.id}}** - ❌ НЕПРАВИЛЬНО
- **Проблема:** Вы используете ID кампании вместо источника трафика
- **Последствие:** В таблице "Джерела трафіку" будут отображаться ID кампаний вместо реальных источников (instagram, facebook, google)
- **Правильно:** `utm_source=instagram` или `utm_source=facebook`

#### 2. **utm_medium=Instagram** - ❌ НЕПРАВИЛЬНО
- **Проблема:** Вы используете название платформы вместо типа канала
- **Последствие:** Неправильная категоризация типа трафика
- **Правильно:** `utm_medium=cpc` (для платной рекламы) или `utm_medium=social` (для органического)

#### 3. **Creo={{placement}}** - ⚠️ НЕ ИСПОЛЬЗУЕТСЯ
- **Проблема:** Это нестандартный параметр, который не сохраняется и не используется в статистике
- **Последствие:** Информация о размещении (placement) теряется
- **Правильно:** Использовать `utm_term={{placement}}` для сохранения информации о размещении

## ✅ Правильный формат для Instagram

### Рекомендуемый формат:
```
utm_source=instagram&utm_medium=cpc&utm_campaign={{campaign.name}}&utm_content={{ad.name}}&utm_term={{placement}}
```

### Полный URL пример:
```
https://twocomms.shop/?utm_source=instagram&utm_medium=cpc&utm_campaign={{campaign.name}}&utm_content={{ad.name}}&utm_term={{placement}}
```

## 📋 Детальное описание каждого параметра

### 1. **utm_source** (ОБЯЗАТЕЛЬНЫЙ) - Источник трафика

**Назначение:** Определяет источник, откуда пришел пользователь

**Правильные значения:**
- `instagram` - для Instagram рекламы
- `facebook` - для Facebook рекламы
- `google` - для Google Ads
- `tiktok` - для TikTok рекламы
- `email` - для email рассылок
- `direct` - для прямого трафика

**Как используется в статистике:**
- Группировка в таблице **"Джерела трафіку (Top 20)"**
- Показывает количество сессий, конверсий, CR%, доход по каждому источнику
- Используется для фильтрации кампаний

**❌ НЕ используйте:**
- `{{campaign.id}}` - это ID кампании, а не источник
- `{{campaign.name}}` - это название кампании
- Любые динамические параметры Meta

**✅ Правильно:**
```
utm_source=instagram
```

---

### 2. **utm_medium** (ОБЯЗАТЕЛЬНЫЙ) - Тип канала

**Назначение:** Определяет тип трафика/канал

**Правильные значения:**
- `cpc` - платная реклама (Cost Per Click)
- `social` - органический трафик из социальных сетей
- `organic` - органический поиск
- `email` - email рассылки
- `referral` - реферальный трафик
- `display` - баннерная реклама

**Как используется в статистике:**
- Сохраняется в базе данных
- Может использоваться для дополнительной фильтрации
- Отображается в детальной информации о сессии

**❌ НЕ используйте:**
- `Instagram` - это название платформы, а не тип канала
- `Facebook` - это название платформы
- Названия платформ

**✅ Правильно:**
```
utm_medium=cpc  # для платной рекламы
utm_medium=social  # для органического трафика
```

---

### 3. **utm_campaign** (РЕКОМЕНДУЕТСЯ) - Название кампании

**Назначение:** Идентифицирует конкретную рекламную кампанию

**Правильные значения:**
- Название кампании из рекламного кабинета
- Можно использовать динамический параметр: `{{campaign.name}}`
- Примеры: `summer_sale_2025`, `new_collection`, `black_friday`

**Как используется в статистике:**
- Группировка в таблице **"Кампанії (Top 20)"**
- Показывает статистику по каждой кампании: сессии, конверсии, CR%, доход
- Используется для фильтрации креативов (utm_content)
- Используется для A/B тестирования креативов

**✅ Правильно:**
```
utm_campaign={{campaign.name}}
# или статическое значение
utm_campaign=summer_sale_2025
```

---

### 4. **utm_content** (ОПЦИОНАЛЬНО) - Креатив/Объявление

**Назначение:** Идентифицирует конкретный креатив или объявление

**Правильные значения:**
- ID объявления: `{{ad.id}}`
- Название объявления: `{{ad.name}}`
- Идентификатор креатива: `creative_video_1`, `banner_top`

**Как используется в статистике:**
- Используется для **A/B тестирования креативов**
- Позволяет сравнить эффективность разных креативов в рамках одной кампании
- Показывает: сессии, конверсии, CR%, доход, средний чек, confidence level
- Определяет победителя A/B теста

**✅ Правильно:**
```
utm_content={{ad.name}}
# или
utm_content={{ad.id}}
```

---

### 5. **utm_term** (ОПЦИОНАЛЬНО) - Дополнительная информация

**Назначение:** Хранит дополнительную информацию (ключевые слова, размещение, аудиторию)

**Правильные значения:**
- Размещение объявления: `{{placement}}`
- Тип аудитории: `lookalike_1`, `interest_sports`
- Ключевое слово: `running_shoes`

**Как используется в статистике:**
- Сохраняется в базе данных
- Отображается в детальной информации о сессии
- Может использоваться для дополнительного анализа

**✅ Правильно:**
```
utm_term={{placement}}
# или
utm_term=feed
utm_term=stories
utm_term=reels
```

---

## 📊 Как параметры используются в статистике Диспетчера

### Таблица "Джерела трафіку (Top 20)"
- **Группировка:** по `utm_source`
- **Показывает:** Сессии, Конверсии, CR%, Дохід, Сер. чек
- **Важно:** Если `utm_source` неправильный (например, ID кампании), статистика будет некорректной

### Таблица "Кампанії (Top 20)"
- **Группировка:** по `utm_campaign` и `utm_source`
- **Показывает:** Сессии, Конверсии, CR%, Дохід
- **Фильтрация:** Можно фильтровать по источнику (`utm_source`)

### A/B тестирование креативов
- **Группировка:** по `utm_content` в рамках одной `utm_campaign`
- **Показывает:** Сравнение креативов, определение победителя, confidence level
- **Важно:** Требует правильного заполнения `utm_campaign` и `utm_content`

### Детальная информация о сессии
- **Показывает:** Все UTM параметры (`utm_source`, `utm_medium`, `utm_campaign`, `utm_content`, `utm_term`)
- **Использование:** Для анализа конкретной сессии пользователя

## 🔧 Динамические параметры Meta (Instagram/Facebook)

Meta поддерживает следующие динамические параметры, которые автоматически заменяются на реальные значения:

- `{{campaign.name}}` - название кампании
- `{{campaign.id}}` - ID кампании (НЕ используйте в utm_source!)
- `{{adset.name}}` - название группы объявлений
- `{{adset.id}}` - ID группы объявлений
- `{{ad.name}}` - название объявления
- `{{ad.id}}` - ID объявления
- `{{site_source_name}}` - источник трафика (facebook, instagram, messenger)
- `{{placement}}` - размещение объявления (feed, stories, reels, etc.)

**⚠️ ВАЖНО:** 
- `{{site_source_name}}` можно использовать в `utm_source`, но лучше указать статическое значение (`instagram` или `facebook`)
- `{{placement}}` используйте в `utm_term`, а не в нестандартном параметре `Creo`

## 📝 Чек-лист для проверки

- [ ] `utm_source` содержит название источника (instagram, facebook, google), а не ID кампании
- [ ] `utm_medium` содержит тип канала (cpc, social, organic), а не название платформы
- [ ] `utm_campaign` содержит название кампании (можно использовать `{{campaign.name}}`)
- [ ] `utm_content` содержит идентификатор креатива (можно использовать `{{ad.name}}` или `{{ad.id}}`)
- [ ] `utm_term` используется для дополнительной информации (например, `{{placement}}`)
- [ ] Удалены нестандартные параметры (например, `Creo`)

---

## 🚨 Последствия неправильного заполнения

Если UTM-теги заполнены неправильно:

1. **Некорректная статистика по источникам** - в таблице "Джерела трафіку" будут отображаться ID кампаний вместо реальных источников
2. **Невозможность сравнения источников** - нельзя будет понять, какой источник (Instagram, Facebook) более эффективен
3. **Потеря данных о размещении** - информация о `{{placement}}` не будет сохраняться, если использовать нестандартный параметр `Creo`
4. **Проблемы с A/B тестированием** - некорректная группировка креативов

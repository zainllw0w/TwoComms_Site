# PDP Content — Critical Findings

**Дата**: 2026-05-16
**Pages analyzed**: 12 PDP (~18% of catalog)
**Источник**: live `https://twocomms.shop/product/...`

## Резюме

Каждая PDP содержит **~1100-1220 слов** видимого текста, но **80-82%** этого текста — **дублируется** между всеми товарами одной категории. Только **~200-300 слов уникального контента на товар**.

Это **главный SEO-провал каталога** и **главная цель этой spec**.

**Шаблонная часть** (~900 слов общих на каждом PDP):
- Блок "Доставка та оплата" (~200 слов) — почти идентичен на всех
- Блок "Догляд та прання" (~150 слов) — идентичен на всех
- Блок "Розмірна сітка / Як обрати розмір" (~150 слов) — идентичен на всех
- Блок "Чому варто купити в TwoComms" (~100 слов) — идентичен
- Tabs FAQ (~150 слов) — частично уникальный, частично шаблон
- Trust badges, navigation, footer copy — на всех

**Уникальная часть** (~200-300 слов):
- H1 (название товара)
- 3-5 предложений короткого описания
- Иногда `details_text` или `target_audience` (опционально)
- 2-3 product FAQ items если заполнены

## Cross-page overlap (по shingle-hashes 5-grams)

| Pair | Overlap |
|---|---|
| business-money ↔ in-shee | **82.1%** |
| hoodie-classic ↔ where-mi-present-hd | **81.9%** |
| 225-tshirt ↔ business-money | **81.6%** |
| in-shee ↔ pokrovsk-girl | **80.4%** |
| classic-tshirt ↔ business-money | **80.3%** |

40+ пар PDP с overlap'ом >70%. **Реальная уникальность контента на типичную PDP: 18-30%**.

Это критично: Google's near-duplicate detection снижает рейтинг страниц при overlap > 30-40% (порог не публичный, но эмпирически известный).

## Что есть сейчас в коде PDP

Из `templates/pages/product_detail.html` и `views/product.py`:

- Title — уникальный (название товара)
- Meta description — **частично уникальный**, fallback на шаблон если seo_description пуст
- `product.short_description` или `product.full_description` — заполнен ВРУЧНУЮ в админке (не у всех)
- `product.details_text` — опциональный
- `product.target_audience` — опциональный, fallback на шаблонный список из 5 пунктов
- `product_faq_items` — из `product.faqs`, **836 items на 65 товаров = ~13 на товар**, но **шаблонные** (не уникальные)
- Tabs: Опис / Розмірна сітка / Доставка / Догляд / FAQ — **все статичные**

## Что нужно сделать (per-product unique content)

### Уровень 1: данные о товаре (есть в БД)

Каждый товар имеет (из модели Product):
- `title` — название
- `short_description`, `full_description`, `description` — копи (часто пустые)
- `details_text` — детали (опционально)
- `target_audience` — целевая аудитория
- `category` (FK) — Tshirt / Hoodie / Longsleeve
- `price`, `discount_percent`, `final_price`
- `is_featured`, `is_new`, `is_dropship_available`
- `material`, `slug`
- `seo_title`, `seo_description`, `seo_keywords`
- `ai_generation_enabled`, `ai_keywords`, `ai_description`, `ai_content_generated`
- Связи: `images`, `faqs`, `color_variants` (с цветами и размерами и stock)
- `average_rating`, `total_reviews`

### Уровень 2: что генерировать ДИНАМИЧЕСКИ (без БД-изменений)

Из существующих данных можно вытянуть:

1. **Уникальный SEO-текст 600-1000 слов на товар** — генерируется по template-функции которая:
   - Берёт title и парсит ключевые токены
   - Берёт цвета из `color_variants` (например 'чорний', 'хакі', 'олива')
   - Берёт размеры из размерной сетки товара
   - Берёт категорию (футболка / худі / лонгслів)
   - Берёт fit-options (regular / oversize)
   - Берёт material (бавовна / трьохнитка)
   - Конструирует текст по структурированному template'у с уникальными вкраплениями про конкретный принт/тему

2. **Уникальный keyword-set на товар** — генерируется как:
   - Базовый набор: `{title.tokens} + {category} + {colors} + {sizes} + {fit} + 'купити' + 'україна'`
   - Long-tail: `'купити {title} в {city}'`, `'{title} на {color}'`, `'{category} з принтом {topic}'`
   - Бренд-тейл: `'{title} TwoComms'`, `'TwoComms {category}'`

3. **Уникальные H2/H3 разделы**:
   - "Кому підійде {title}" — описание ЦА на основе category + topic
   - "Як стилізувати {title}" — стилистический совет на основе category + цвета
   - "Догляд за {title}" — общий гайд + специфика по material + DTF
   - "Доставка {title} по Україні" — генерируется из delivery policy + size
   - "Чому варто купити {title}" — preposition list

4. **Уникальный per-product FAQ** (заменить нынешние шаблонные):
   - "Чи дає усадку {title} після прання?" — answer на основе material
   - "Які кольори доступні для {title}?" — answer из color_variants
   - "Як обрати розмір {title}?" — answer из size_chart + fit
   - "Чи є {title} в чоловічому/жіночому розмірі?" — answer
   - "Скільки коштує доставка {title}?" — answer из weight class

### Уровень 3: что хранить в БД (новые поля или обогащение)

Чтобы каждый товар имел реально уникальный текст про принт/тему — нужно поле `topic_narrative` или `print_story`:
- Для `business-money`: «Принт "Бізнес — це математика" — про підприємницький дух, про те що бізнес це не магія а формула. Чисті лінії, мінімалістичний шрифт.»
- Для `pokrovsk-girl`: «Принт "Дівчина з Покровська" — символ стійкості жінок прифронтового міста. Графічний стиль з військовими акцентами.»
- Для `kharkiv-district`: «Принт "Харківська область" — карта з контурами, про любов до регіону. Для тих хто з Слобожанщини.»

Это поле — **единственное что нужно вручную наполнять** для каждого товара. Всё остальное (стиль, доставка, материалы, размеры) генерируется автоматически.

### Архитектура решения

Создать новый template tag/service:
- `storefront/services/product_seo_block.py` — генератор уникального SEO-блока для PDP
- Принимает `product` + `language_code`
- Возвращает контейнер с разделами: hero_intro / who_for / how_to_style / care / delivery / why_us / faq
- Каждый раздел берёт template-формулу и подставляет реальные данные товара
- Добавляет inline-ссылки на связанные товары (триплет, related), категорию, гайды

И поле в БД (через миграцию):
- `Product.topic_narrative` — TextField, опциональный
- `Product.topic_keywords` — CharField (через запятую), опциональный

Если `topic_narrative` пуст — fallback на template-генерацию из title.

### Мультиязычность

Все генераторы должны работать через `gettext_lazy` или иметь отдельные template-формулы для UA/RU/EN. Поле `topic_narrative` — `modeltranslation` (как уже сделано для `title` и других).

## Финальный SEO impact

Если довести overlap до **<30%** на каждой PDP:
- Google перестаёт считать страницы near-duplicate → больше strаницы попадают в индекс с глубокой индексацией
- Растёт количество long-tail запросов на которые сайт ранжируется (с 65 товаров можно получить 200-400 long-tail rankings)
- AI search (ChatGPT/Perplexity/Claude) получает уникальный материал для цитирования каждого товара отдельно
- AI Overviews и Bing Copilot цитируют конкретный товар при запросе «футболка з принтом про харків» а не общий каталог

## Связь с другими аудитами

- **Schema audit (02)**: per-product Schema нужно расширить полями `category`, `color`, `size`, `material`, `audience`.
- **Linking audit (04)**: новый блок должен включать smart inline links на триплеты и related products.
- **Keywords audit (06)**: нужно собрать keyword universe для всех тем принтов (харків, бізнес, війна, побут, культура).
- **Tech debt (05)**: текущие шаблонные блоки в `product_detail.html` (доставка, догляд, FAQ) можно заменить на динамическую генерацию.

# 🕐 Инструкция: Настройка CRON через cPanel

## ❓ Как работает CRON?

**CRON работает автоматически** - это система на сервере, которая запускает команды по расписанию.

- ✅ **После настройки - работает САМ, без вашего участия**
- ✅ **Не нужно запускать вручную**
- ✅ **Не нужно подключаться по SSH каждый раз**
- ✅ **Работает 24/7 на сервере**

**Простыми словами:** Вы настраиваете один раз через cPanel, и сервер сам выполняет команду по расписанию (каждые 5 минут, каждый час, и т.д.)

---

## 📋 ТЕКУЩЕЕ СОСТОЯНИЕ

**У вас УЖЕ настроен cron для Nova Poshta:**
- ✅ Работает **каждые 15 минут** (`*/15 * * * *`)
- ⚠️ Нужно изменить на **каждые 5 минут** (`*/5 * * * *`)

## 🎯 НАСТРОЙКА ЧЕРЕЗ CPANEL (БЕЗ SSH)

### Шаг 1: Войти в cPanel

1. Откройте cPanel вашего хостинга
2. Найдите раздел **"Дополнительно" (Advanced)** или **"Задачи Cron" (Cron Jobs)**

### Шаг 2: Добавить/Изменить задачу

1. Найдите существующую задачу для `update_tracking_statuses`
2. Если есть - **отредактируйте** её
3. Если нет - **добавьте новую**

### Шаг 3: Заполнить форму

#### Поля в cPanel:

**Время выполнения:**
- Минута: `*/5` (каждые 5 минут)
- Час: `*` (каждый час)
- День: `*` (каждый день)
- Месяц: `*` (каждый месяц)
- День недели: `*` (каждый день недели)

**Команда для выполнения:**

```bash
cd /home/qlknpodo/TWC/TwoComms_Site/twocomms && /home/qlknpodo/virtualenv/TWC/TwoComms_Site/twocomms/3.14/bin/python manage.py update_tracking_statuses >> /home/qlknpodo/TWC/TwoComms_Site/twocomms/logs/nova_poshta_cron.log 2>&1
```

Или более короткий вариант (если лог в другом месте):

```bash
cd /home/qlknpodo/TWC/TwoComms_Site/twocomms && /home/qlknpodo/virtualenv/TWC/TwoComms_Site/twocomms/3.14/bin/python manage.py update_tracking_statuses >> /tmp/nova_poshta_update.log 2>&1
```

## 📝 ВАРИАНТ 1: Через интерфейс cPanel (Рекомендуется)

### Где найти Cron Jobs в cPanel:

1. **Вариант A:** Дополнительно → Cron Jobs
2. **Вариант B:** Advanced → Cron Jobs  
3. **Вариант C:** Поиск в cPanel: "Cron" или "Запланированные задачи"

### Что делать:

1. Нажмите **"Cron Jobs"** или **"Запланированные задачи"**
2. Прокрутите вниз до раздела **"Добавить новую задачу Cron"** или **"Стандарт (Advanced)"**
3. Выберите **"Стандарт (Advanced)"** - там можно указать точное время

4. Заполните поля:

```
Минута:  */5
Час:     *
День:    *
Месяц:   *
Неделя:  *
```

5. В поле **"Команда"** вставьте:

```bash
cd /home/qlknpodo/TWC/TwoComms_Site/twocomms && /home/qlknpodo/virtualenv/TWC/TwoComms_Site/twocomms/3.14/bin/python manage.py update_tracking_statuses >> /home/qlknpodo/TWC/TwoComms_Site/twocomms/logs/nova_poshta_cron.log 2>&1
```

6. Нажмите **"Добавить новую задачу Cron"** или **"Сохранить"**

---

## 📝 ВАРИАНТ 2: Обновить существующую задачу

Если задача уже есть (каждые 15 минут):

1. Найдите её в списке существующих задач
2. Нажмите **"Изменить"** или **"Edit"**
3. Измените **Минута:** с `*/15` на `*/5`
4. Нажмите **"Сохранить"**

## ⚙️ РАСПИСАНИЕ (Расшифровка)

### `*/5 * * * *` означает:

- `*/5` - каждые 5 минут (00, 05, 10, 15, 20, 25, 30, 35, 40, 45, 50, 55)
- `*` - каждый час
- `*` - каждый день месяца
- `*` - каждый месяц
- `*` - каждый день недели

**Итого:** Команда запускается каждые 5 минут, 24 часа в сутки, 7 дней в неделю.

## 🆘 ПРОБЛЕМЫ И РЕШЕНИЯ

### Задача не выполняется:

1. **Проверьте права:** Убедитесь что путь к Python правильный
2. **Проверьте логи:** Посмотрите ошибки в лог-файле
3. **Проверьте API ключ:** Убедитесь что `NOVA_POSHTA_API_KEY` настроен

### Логи не создаются:

1. Проверьте что директория существует:
   ```bash
   mkdir -p /home/qlknpodo/TWC/TwoComms_Site/twocomms/logs
   ```

2. Используйте другой путь для лога (например `/tmp/nova_poshta_update.log`)

# TwoComms Instagram AI-агент — дизайн і дорожня карта

Дата: 2026-06-15. Статус: Фаза 0 реалізована; Фази 1–2 чекають контенту/аппрува.

## Мета
Перетворити Direct-бота на повноцінного AI-агента бренду: знає бренд, каталог,
склад, ціни; підбирає розмір; веде до замовлення; створює оплату Monobank;
валідує Нову Пошту; працює швидко, безпечно, без інʼєкцій і спаму.

## Архітектура контексту (вже працює)
У `system_instruction` Gemini інжектиться в порядку пріоритету:
1. **Guardrails-промпт** (роль, анти-інʼєкція, не розкривати модель, не вигадувати).
2. **Оперативні директиви** — поле `knowledge_base` в адмінці (миттєво, найвищий
   пріоритет). Напр.: «немає футболок з резинкою → пропонувати без резинки».
3. **База знань** — Markdown-файли `management/bot_knowledge/*.md` (git, кеш за mtime).
4. **Каталог** — товари з БД (ціни/наявність/кольори/URL), кеш 10 хв.

### Куди класти знання (відповідь на питання «який файл/формат»)
- Постійні знання про бренд → редагуй `twocomms/management/bot_knowledge/brand.md`
  (Markdown, секції вже заготовлені). Можна додати ще `.md` файли в цю теку —
  усі підхопляться.
- Оперативні/тимчасові директиви → поле «База знань» на вкладці «Бот» (без коміту).
- Розмірні сітки → секція в `brand.md` (посилання по категоріях) — бот надсилає
  потрібне посилання як текст.

## Фаза 0 — Фундамент знань + анти-абуз (РЕАЛІЗОВАНО)
- Завантажувач `bot_knowledge.py` (MD + кеш за mtime).
- Шаблон `brand.md` із секціями (бренд, засновник, доставка, оплата, повернення,
  промо, розмірні сітки, підбір розміру, колаборації, тон, FAQ).
- Інжект трьох шарів у Gemini.
- Анти-абуз: повтор одного й того ж питання >3 разів за 10 хв → дешева відповідь
  без виклику Gemini (економія токенів). Плюс rate-limit 25/год на відправника.

## Фаза 1 — Read-only function calling (потребує аппрува)
Інструменти Gemini (тільки читання, безпечно, без грошей):
- `get_product(query)` → ціна, наявність, кольори, URL.
- `check_stock(product, color, size)` → є/немає.
- `promo_info(code)` → перевірка промокоду в БД, що дає.
- `size_chart(category)` → посилання на сітку.
Інтеграція з `storefront`/`productcolors`/`warehouse`/promo моделями.

## Фаза 2 — Транзакційний агент (ВИСОКИЙ РИЗИК; потребує аппрува + даних)
- Створення замовлення на сайті за зібраними даними (імʼя, телефон, НП-відділення).
- Валідація Нової Пошти через API (місто/відділення) з перепитом при помилці.
- Прив'язка клієнта до телефону/ПІБ (захист від мульти-акаунтів на промокод).
- Створення посилання на оплату Monobank + перевірка статусу → статус «оплачено»
  лише за фактом від Monobank (не «на думку» бота).
- Сповіщення в Telegram + лід в Instagram (зміна категорії контакту в бізнес-акаунті).
- Ескалація менеджеру (вже є тег `[MANAGER]`).

### Ризики/застереження Фази 2
- ШІ не повинен довільно міняти статуси виконання («відправлено») — лише людина/ТТН.
- «Оплачено» — лише за об'єктивним статусом Monobank.
- Потрібен App Review (Advanced Access на `instagram_business_manage_messages`),
  щоб відповідати всім; зараз — лише ролям/тестувальникам.
- Кожен крок — окремі функції + тести (TDD), без техборгу.

## Що потрібно від власника
1. Заповнити `brand.md` (бренд, засновник, доставка, оплата, промо-політика, FAQ).
2. Дати посилання на розмірні сітки (по категоріях).
3. Підтвердити правила промокодів і логіку замовлення/оплати для Фази 2.
4. Пройти App Review для відповідей усім користувачам.

# Редизайн статистики менеджмента — дизайн

> Дата: 2026-06-10 · Скоуп: страница `/stats/` (менеджер + админ-просмотр) и список
> менеджеров `/stats/admin/`. Источник истины — код в `twocomms/management/`.

## 1. Проблема

Текущая `stats.html` технически богатая, но **перегруженная**: на одном экране
спираль+КПД, гигантский MOSAIC-showcase (радар, waterfall, симулятор зарплаты,
телефония, апелляции, готовность, критично/возможности, rescue, прогноз,
таймлайн), 8 KPI-карточек, советы и ещё 6 линейных секций. Нет иерархии
«главное vs второстепенное». Часть данных мёртвая/нулевая:

- **Телефония/прослушивание** — в проде нет webhook, `CallRecord` пуст → всегда
  нули и `DEGRADED`. Пользователь прямо просил убрать.
- **DTF-мост** — всегда degraded/пусто (на admin-панели, не на stats).
- **Апелляции** — `delta_score` всегда 0 («театр»).
- **Симулятор зарплаты / прогноз** — `shadow`, на выплаты не влияют.

MOSAIC — `shadow`-рейтинг, строится on-demand; на периодах без snapshot блок
пустой.

## 2. Цель

Кардинально красивый, чёткий, интерактивный дашборд **без перегруза**: ровная
bento-сетка, явная иерархия показателей, MOSAIC и КПД поданы крупно и красиво,
графики/кольца/гейджи/бары с анимацией. Единый вид для менеджера и для
админ-просмотра. Разграничение доступа сохраняется без изменений.

## 3. Жёсткие ограничения (тесты)

UI-регрессии (`tests_phase3_snapshots`, `tests_phase4_analytics`) при наличии
snapshot требуют на `/stats/` наличие строк (по сырому HTML, видимость не важна):
`MOSAIC`, `SHADOW` (из json_script-дампа payload), и заголовков: «Що змінилося
сьогодні», «Критично на сьогодні», «Найкращі можливості», «Топ-5 на порятунок»,
«Симулятор винагороди», «Радарний огляд», «Таймлайн комунікації з клієнтом»,
«Відкрити розклад», «Подати апеляцію». `forecast/readiness/incidents`
проверяются только на `/admin-panel/` (не трогаем).

**Следствие:** глубокие shadow-блоки нельзя удалять из DOM. Решение —
**прогрессивное раскрытие**: они остаются в разметке, но сворачиваются в
закрытый по умолчанию `<details>`. `assertContains` зелёный, первый экран чистый.
Физически удаляется только карточка **телефонии** (тестами не покрыта).

## 4. Архитектура UI (bento + прогрессивное раскрытие)

Подход: единая адаптивная **bento-grid** для верхних уровней + один блок деталей
с сегмент-переключателем (табы) + свёрнутая «Поглиблена аналітика (тінь)».

### Зона 0 — Шапка
Заголовок, период-лейбл, фильтры (Сьогодні/Вчора/Тиждень/Місяць + диапазон).
Для админа — чип «Перегляд: Имя» + «← До списку».

### Зона 1 — HERO (главное, виден сразу)
- **MOSAIC** — крупное радиальное кольцо: значение по центру (count-up), цвет
  по зоне (красный <40 / янтарь 40–70 / зелёный >70), дельта vs прошлый период,
  полоска довіри (confidence band), мини-степпер пейса (мінімум/ціль/відновлення).
  Без snapshot — нейтральное «Накопичуємо дані» + сколько обработок нужно.
- **КПД** — крупное число + дельта + segmented-bar разбивки (зусилля/якість/
  операції/штраф) + `kpd_insight` строкой.
- **4 ключевых KPI** — Бали, Клієнти, Активний час, Передзвони (крупно, компактно).
- **«Критично на сьогодні»** (`must_do_today`) — actionable список (просрочені
  передзвони, контакти магазинів, клієнти без плану). Пусто → «Все під контролем».

### Зона 2 — Важное (bento)
- **Дисципліна** — полукруг-gauge пропусков передзвонів + mini (виконано/
  перенесено/скасовано/відкрито) + опоздания отчётов + сигнал «без плану N».
- **Розподіл результатів** — донат/спіраль `segments` (call_result) + легенда.
- **Поради** (`advice`) — 3 на виду, tone-цвета, «ще N», пометка «припущення».
- **Тренди** — 3 спарклайна (Бали/КПД/Активність) с area-заливкой и точками.
- **Найкращі можливості** (`best_opportunities`).

### Зона 3 — Деталі (табы, по умолчанию «Джерела»)
Сегмент-переключатель: Джерела · Магазини · Накладні · КП e-mail · Звітність.
Контент — бары источников, mini-сетки магазинов/накладных/КП, timeline отчётов.

### Зона 4 — Поглиблена аналітика (тінь), `<details>` свёрнут
Радарний огляд, Що змінилося сьогодні, Симулятор винагороди, Прогнозні
діапазони, Карта готовності, Топ-5 на порятунок, Таймлайн комунікації, кнопки
«Відкрити розклад» / «Подати апеляцію» + drawers. Рендерится только при snapshot.

## 5. Визуализации и интерактив
Кольца (MOSAIC), полукруг-gauge (дисципліна), спарклайны с area (тренди), донат
(результати), радар (осі MOSAIC), горизонтальні бари (джерела, розбивка КПД).
Анимации: count-up чисел (IntersectionObserver), заполнение дуг/баров, hover-
подсветка+тултипы, переключение табов без перезагрузки. Учитывается
`prefers-reduced-motion`. Пустые состояния — аккуратные, без «нулей повсюду».

## 6. Админ vs менеджер
Единый `stats.html`. Менеджер — всегда `user=request.user`. Админ (`is_staff`) —
любой через `user_id`, видит ту же красивую страницу + контекст-бар. Список
`/stats/admin/` — редизайн в карточки команды (имя, роль, online-пульс,
последний вход) в едином bento-стиле. Доступ и JSON-эндпоинты (403 для чужих) —
**без изменений**.

## 7. Скоуп правок (минимизация риска)
- `templates/management/stats.html` — новая структура.
- `templates/management/components/shadow_showcase.html` — реструктуризация,
  удаление карточки телефонии, обёртка глубокого в `<details>`.
- `templates/management/stats_admin_list.html` — редизайн карточек.
- `static/css/management-stats.css` — переписать на bento-систему (отдельный
  файл, грузится прямым `{% static %}`, не через styles.css и не в `{% compress %}`).
- `static/js/management-stats.js` — добавить кольцо MOSAIC, segmented-bar КПД,
  count-up, табы, area-спарклайны; сохранить существующие render-функции и drawers.
- **Бэкенд `stats_service.py`/`stats_views.py` — не меняется** (все производные
  считаются в JS из json_script payload).

## 8. Тестирование и деплой
1. `management.tests_phase3_snapshots`, `tests_phase4_analytics`,
   `tests_template_regressions` (целевые для правок).
2. Полный `python manage.py test management`.
3. Локальный рендер страниц (проверка шаблонов и обязательных строк).
Деплой по чек-листу: `collectstatic --no-input` + `compress --force` +
`touch tmp/restart.txt`. Миграций нет.

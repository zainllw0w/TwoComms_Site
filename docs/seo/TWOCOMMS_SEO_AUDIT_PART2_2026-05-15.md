## 8. СЕМАНТИЧНЕ ЯДРО — АНАЛІЗ

### 8.1 Поточна реалізація (seo_utils.py)

Семантичне ядро формується автоматично через `SEOKeywordGenerator`:

**BASE_KEYWORDS (15):** `стріт одяг, мілітарі одяг, футболки, худі, лонгсліви, ексклюзивний дизайн, якісний одяг, одяг з характером, стріт стиль, мілітарі стиль, TwoComms, купити одяг онлайн, модний одяг, трендовий одяг, український бренд`

**Проблеми:**
1. ⚠️ **Відсутні LSI-кластери:** Немає синонімів і long-tail варіацій. Google очікує семантичну глибину.
2. ⚠️ **Відсутні geo-ключі в ядрі:** "Харків", "Україна", "Київ" — тільки в bottom-copy каталогу.
3. ⚠️ **Немає intent-based кластеризації:** "купити", "замовити", "ціна" змішані з інформаційними.
4. ⚠️ **Meta keywords tag:** Google його ігнорує з 2009. Витрати на генерацію — марні.

## 10. SEO-COCON (ТЕМАТИЧНИЙ КОКОН)

### 10.1 Що таке SEO-Cocon?

Французька SEO-методологія (Laurent Bourrelly): створення щільної мережі внутрішніх посилань навколо тематичних кластерів, де кожна "дочірня" сторінка підсилює "батьківську" через контекстні посилання.

### 10.2 Поточний стан внутрішньої перелінковки

**Вже реалізовано:**
- ✅ Footer з 4 колонками навігації (Покупка, Підтримка, Бренд, Швидкий доступ)
- ✅ Cross-links між support pages (delivery↔faq↔returns↔help_center)
- ✅ Category deep links на delivery, size_guide, returns pages
- ✅ PDP → size_guide, delivery, returns (trust badges)
- ✅ Catalog → custom_print (CTA панель)
- ✅ Related products на PDP (до 8 товарів)

**Відсутнє:**
- ❌ Blog/Content hub (інформаційні статті)
- ❌ Look-book або style guide сторінки
- ❌ Тематичні landing pages (наприклад, "Одяг для ветеранів", "Streetwear для чоловіків")
- ❌ Internal linking між товарами одного дизайну (ts↔hd↔ls)

### 10.3 Рекомендована SEO-Cocon архітектура

```
                    ГОЛОВНА
                   /   |    \
              Каталог  Бренд  Підтримка
             /  |  \          /  |  \  \
          TS   HD   LS     FAQ  Del  Ret  Help
          |    |    |
       [товари одного дизайну — cross-linked]
```

**Ключова рекомендація:** Створити cross-links між варіантами одного дизайну:
- На PDP "Футболка «My Little Baby»" додати: "Також доступно: [Худі](link) | [Лонгслів](link)"
- Це створить тематичний кокон навколо кожного дизайну

### 10.4 Що дасть SEO-Cocon?

1. **+15-25% internal PageRank flow** до category та PDP pages
2. **Зменшення bounce rate** — користувачі переходять між варіантами
3. **Краще розуміння топіків Google** — семантичні кластери замість ізольованих сторінок
4. **Long-tail ranking** — "купити худі my little baby twocomms" замість тільки "худі twocomms"

## 11. КОНТЕНТ — ЯКІСТЬ ТЕКСТІВ

### 11.1 PDP Description

**Поточний шаблон (product_detail.html:397-398):**
```html
{{ title }} — це не просто одяг. Це стан.
Створений для тих, хто йде своїм шляхом...
```

**Проблема:** Цей текст ОДНАКОВИЙ для всіх 65 товарів. Google класифікує як thin content.

**Рекомендація:** Для топ-20 товарів написати унікальний lead paragraph (2-3 речення).

### 11.2 Шаблонні деталі

Якщо `product.details_text` порожній, показується шаблон:
```
Матеріал: 95% бавовна, 5% еластан — преміум якість, 190 г/м²
```

Це однаково для футболок, худі та лонгсліві, хоча реальні параметри різні (худі — 320 г/м² трьохнитка).

### 11.3 Bottom SEO copy (каталог)

**Позитив:** Каталог має 3 H2 з geo-ключами та комерційним intent — ✅ добра робота.

**Проблема:** Текст містить hardcoded міста без реальної geo-релевантності (немає фізичних точок у Львові/Дніпрі).

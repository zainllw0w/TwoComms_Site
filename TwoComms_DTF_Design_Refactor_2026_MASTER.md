# 1. Контекст бизнеса DTF (почему обычный дизайн тут не работает)

## 1.1 Что покупает клиент DTF на самом деле
Не «печать», а:
- **предсказуемый результат**
- **отсутствие косяков**
- **скорость**
- **контроль**
- **уверенность, что файл не убьют**

Следовательно:
> UI должен выглядеть как **производственный инструмент**,  
> а не как маркетинговый лендинг.

---

## 1.2 Ключевые страхи клиента
- «Файл плохой — напечатают как есть»
- «Я неправильно посчитал метры»
- «На чёрной ткани будет плохо»
- «Затянется по срокам»
- «Мне потом не ответят»

**Каждая визуальная фича должна либо:**
- снижать страх  
- либо усиливать ощущение контроля  
- либо ускорять решение  

Если нет — она лишняя.

# 2. Глобальная дизайн‑система (Design DNA)

## 2.1 DTF Narrative System (обязательный каркас)

Весь сайт подчиняется скрытой истории:

**Intake → Preflight → Print → Powder → Cure → Peel → Pack → Ship**

Это не сторителлинг ради красоты, а:
- снижение тревожности,
- объяснение цены,
- ощущение технологического процесса.

---

## 2.2 Три семейства эффектов (жёсткое правило)

### SCAN
- линии
- прицелы
- registration marks
- rulers
- G‑code траектории

**Смысл:** точность, контроль, инженерия.

---

### HEAT
- orange / red glow
- thermal‑подсветка
- cutoff‑таймеры

**Смысл:** скорость, производство, горячий процесс.

---

### PARTICLES
- порошок
- капли чернил
- микропыль

**Смысл:** физичность материала.

> ❗ На одном экране — ОДНО доминирующее семейство.

---

## 2.3 Material System (молекулярный уровень)

### PET‑film
- микроблики
- едва заметные царапины
- ощущение тонкого пластика

### Powder
- зерно 2–4%
- оседание при idle

### Carbon / Metal
- матовость
- структурный шум

**Важно:**  
на формах и тексте — почти 0%.

# 3. HOME PAGE — глубокий разбор

## 3.1 Hero: Printhead Scan

### Что это
Имитация проходов печатающей головы, которая:
- проявляет заголовок
- подсвечивает цену
- активирует CTA

### Почему это идеально для DTF
- это **прямая метафора процесса**
- мозг моментально понимает: «это производство»

### Риски
- WebGL может убить слабые устройства

### Решение
- Desktop: Canvas / WebGL (Tier 2–3)
- Mobile: SVG mask + CSS
- Reduce motion: статичный reveal

### Влияние
- Визуал: 10/10
- Запоминаемость: 10/10
- Конверсия: + при сохранении простого CTA

---

## 3.2 Ink / Powder Particles

### Где можно
- hero background
- storytelling блоки

### Где нельзя
- Order
- формы
- pricing таблицы

### Улучшение
- не continuous animation
- обновление раз в 2–3 секунды

---

## 3.3 Hero‑структура (идеальная)

1. Заголовок (короткий, технический)
2. Цена + диапазон
3. CTA
4. Micro‑HUD:
   - 60cm
   - Hot Peel
   - QC check
   - 24–48h

# 4. STORYTELLING / PROCESS

## 4.1 Scroll‑story «4 шага = 4 машины»

### Польза
- объясняет технологию
- оправдывает цену
- снижает страх

### Риск
- слишком длинный скролл убивает CR

### Решение
- 1 экран = 1 шаг
- CTA после каждого шага

---

## 4.2 Peel Reveal (критический анализ)

### Где идеально
- объяснение технологии
- галерея

### Где опасно
- повторять часто → выглядит как gimmick

## 5.1 Главная проблема рынка

- просто картинки
- нет доказательства качества

---

## 5.2 Must‑have элементы

- film → fabric

### Macro‑loupe
- растр
- тонкие линии
- микро‑текст

### QC‑stamp
- «QC PASSED»

### Паспорт кейса

## 5.3 Психология

Если тут нет доказательств — клиент идёт писать.

# ITER2 — dtf.twocomms.shop: тексты + UI микропак + промпт исполнителю

Дата: 2026‑02‑22  
Контекст: 3‑я итерация сайта; цель — убрать «ИИ‑тон» и технический жаргон, повысить понятность для холодного Instagram‑трафика, при этом не ломая текущую структуру, графику и анимации.

## Что именно вы получите в этих файлах

1) **Решения и приоритеты (P0/P1/P2)** — что меняем прямо сейчас, что позже, что исключаем как риск/оверкилл.  
2) **Точные тексты для всех ключевых страниц** (UA + RU + EN) — включая CTA, microcopy, ошибки форм, статусы «перевірки файлу».  
3) **UI‑микропакет** — точечные микро‑анимации/иконки (лампочка «є світло», сканирование файла, подсветка CTA), с правилами по производительности и `prefers-reduced-motion`.  
4) **Промпт для 3‑го агента‑исполнителя** — готовый план правок, критерии приёмки и анти‑рисковые ограничения.

## Главный принцип Iter2
**Меняем слова → уменьшаем когнитивную нагрузку → добавляем “следующий шаг” в нужный момент.**  
Никаких фейковых таймеров/дефицита/«псевдо‑очередей». Если показываем метрику — только реальную.

## Файлы в пакете

- `ITER2_01_DECISIONS.md` — решения, аргументы, что исключаем и почему, чек‑лист готовности.
- `ITER2_02_COPY_UA.md` — финальные тексты (украинская версия) по всем страницам + системные строки.
- `ITER2_03_COPY_RU.md` — финальные тексты (русская версия) по всем страницам + системные строки.
- `ITER2_04_COPY_EN.md` — финальные тексты (английская версия) по всем страницам + системные строки.
- `ITER2_05_UI_MICROPACK.md` — микро‑графика/анимации и правила внедрения без потери скорости.
- `ITER2_06_PROMPT_AGENT3_IMPLEMENT.md` — промпт для агента, который будет править сайт (копирайт + мелкие UI‑изменения).

## Как внедрять (коротко)

1) Исполнитель берёт `ITER2_06_PROMPT_AGENT3_IMPLEMENT.md` как основной промпт.  
2) Тексты копирует **строго** из `ITER2_02/03/04_COPY_*`.  
3) UI‑микропак — по `ITER2_05_UI_MICROPACK.md` (P0 только то, что отмечено P0).  
4) После правок — проход по чек‑листу (в конце файла‑промпта): нет англицизмов в UA/RU, нет дублей в шапке, мобильный dock не дублирует /order/, CTA‑иерархия соблюдена.

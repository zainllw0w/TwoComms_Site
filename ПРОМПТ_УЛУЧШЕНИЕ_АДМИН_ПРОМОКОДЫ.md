# 🎯 ПРОМПТ: Улучшение Админ-Панели Промокодов

## 📋 ЗАДАЧА

Необходимо улучшить интерфейс админ-панели для управления промокодами в Django-проекте TwoComms. Текущий шаблон имеет проблемы с дизайном, функциональностью и удобством использования.

---

## 🔍 КОНТЕКСТ ПРОЕКТА

### Расположение файлов

**Основные файлы:**
- `twocomms/twocomms_django_theme/templates/pages/admin_promocodes.html` - основной шаблон админ-панели промокодов
- `twocomms/storefront/views/promo.py` - views и логика работы с промокодами
- `twocomms/storefront/models.py` (классы `PromoCode`, `PromoCodeGroup`, `PromoCodeUsage`) - модели данных
- `twocomms/storefront/urls.py` - маршруты для промокодов
- `twocomms/twocomms_django_theme/static/css/promocodes.css` - стили промокодов

**Интеграция в админ-панель:**
- Админ-панель промокодов открывается через `/admin-panel/?section=promocodes&tab=<tab_name>`
- Подключается через `twocomms/twocomms_django_theme/templates/pages/admin_panel.html` (строка ~1637: `{% include 'pages/admin_promocodes.html' %}`)

### Модели данных

**PromoCode (Промокод):**
- `code` - код промокода (уникальный)
- `promo_type` - тип (regular/voucher/grouped)
- `discount_type` - тип скидки (percentage/fixed)
- `discount_value` - значение скидки
- `group` - ForeignKey к PromoCodeGroup (может быть NULL)
- `max_uses`, `current_uses` - ограничения использования
- `one_time_per_user` - одноразовое использование на пользователя
- `min_order_amount` - минимальная сумма заказа
- `valid_from`, `valid_until` - период действия
- `is_active` - статус активности

**PromoCodeGroup (Группа промокодов):**
- `name` - название группы
- `description` - описание
- `one_per_account` - **ВАЖНО**: если True, то пользователь, использовавший хотя бы один промокод из группы, больше не сможет использовать другие промокоды из этой группы
- `is_active` - статус активности

### Логика работы

**Групповая логика (критически важно):**
Если группа имеет `one_per_account=True`, то после использования любого промокода из этой группы конкретным пользователем, этот пользователь больше не сможет использовать другие промокоды из этой группы (проверка через `PromoCodeUsage` модель).

**AJAX endpoints:**
- `/admin-panel/promocode/<id>/get-form/` - получение данных промокода для редактирования
- `/admin-panel/promocode/<id>/edit-ajax/` - обновление промокода через AJAX
- `/admin-panel/promocode/<id>/change-group/` - изменение группы промокода (для drag & drop)
- `/admin-panel/promo-group/<id>/get-form/` - получение данных группы
- `/admin-panel/promo-group/<id>/edit-ajax/` - обновление группы через AJAX

## ❌ ТЕКУЩИЕ ПРОБЛЕМЫ

### 1. Проблемы с дизайном
- Fade background перекрывает вкладки модальных окон (z-index конфликты)
- Стили не применяются правильно, элементы "подвинуты" на бок
- Неправильное применение auto-layout
- Некоторые элементы выходят за рамки контейнера

### 2. Проблемы с функциональностью
- Drag & Drop работает, но промокоды и группы находятся в разных вкладках/местах, что усложняет перетаскивание
- Модальные окна имеют проблемы с backdrop (затемнением фона), который перекрывает элементы интерфейса
- Неудобная навигация между вкладками во время drag & drop

### 3. Проблемы с UX
- Неочевидное управление промокодами
- Статистика неудобно расположена
- Редактирование требует много кликов

## 📝 СПЕЦИФИЧЕСКИЕ ТЕХНИЧЕСКИЕ ДЕТАЛИ

### Z-index конфликты
Проблема: backdrop модальных окон (обычно z-index: 1040 в Bootstrap) может перекрывать элементы с меньшим z-index.

**Решение:**
- Проверить z-index всех элементов интерфейса
- Модальные окна должны иметь z-index выше, чем вкладки
- Backdrop должен быть ниже модального окна, но выше основного контента
- Использовать правильную иерархию: контент < backdrop < модальное окно

### Drag & Drop оптимизация

**Текущая реализация:**
- Промокоды на вкладке "Промокоды" (draggable)
- Группы на вкладке "Группы" (drop zones)
- Они в разных местах - неудобно

**Возможные решения:**

**Вариант 1: Split-view на вкладке промокодов**
- Левая часть: список промокодов
- Правая часть: список групп (compact view)
- Можно перетаскивать промокоды в группы справа

**Вариант 2: Визуальные индикаторы на карточках промокодов**
- На каждой карточке промокода показать, в какой группе он находится
- Добавить dropdown для быстрой смены группы
- Оставить drag & drop как дополнительную функцию

**Вариант 3: Улучшенная навигация**
- При начале drag промокода автоматически переключаться на вкладку групп
- Или показывать группы в sidebar во время drag

**Рекомендация:** Реализовать вариант, который будет наиболее удобным для администратора. Возможно, комбинацию вариантов.

### Layout проблемы

**Проблемы:**
- Элементы "подвинуты" на бок
- Неправильное применение auto-layout

**Решение:**
- Проверить все flexbox/grid контейнеры
- Убедиться, что width/max-width установлены правильно
- Проверить margin/padding всех элементов
- Использовать правильные breakpoints для адаптивности

## 🔍 РАЗНИЦА МЕЖДУ LEAD И PURCHASE

### **Визуально в списке:**

```
📋 Lead                    ✅ Purchase
├─ Нет иконки денег       ├─ Есть иконка 💰 + сумма
├─ Светло-серый фон       ├─ Светло-зеленый фон
└─ Меньше параметров      └─ Больше параметров (contents)
```

### **В параметрах:**

| Параметр | Lead | Purchase |
|----------|------|----------|
| **value** | Предполагаемая сумма | Реальная оплаченная сумма |
| **content_name** | ✅ Есть | ❌ Нет |
| **content_category** | ✅ `order_lead` | ❌ Нет |
| **content_type** | ❌ Нет | ✅ `product` |
| **contents** | ❌ Нет | ✅ Список товаров |
| **Advanced Matching** | 4 параметра (без email*) | 5 параметров (с email*) |

*Email только если пользователь авторизован

## 🎓 ПОЛЕЗНЫЕ ТЕРМИНЫ

| Термин | На русском | Что означает |
|--------|-----------|--------------|
| **Event** | Подія | Событие (Lead, Purchase) |
| **Advanced Matching** | Розширене співставлення | Дополнительные данные о пользователе |
| **Test Events** | Тестові події | Режим тестирования событий в реальном времени |
| **Overview** | Огляд | Общая статистика |
| **Event Source** | Джерело події | Откуда пришло событие (Web, App) |
| **Data Source** | Джерело даних | Pixel или другой источник данных |
| **Parameter** | Параметр | Дополнительная информация о событии |

## ✅ ЧЕКЛИСТ: ЧТО ПРОВЕРИТЬ

После тестирования убедитесь что видите:

### **Для Lead Event:**
- [ ] ✅ Название события: `Lead` (не `Purchase`)
- [ ] ✅ Параметр `content_category`: `order_lead`
- [ ] ✅ Есть `value` и `currency`
- [ ] ✅ Advanced Matching: минимум 4 параметра
- [ ] ✅ Event Source: `Web`

### **Для Purchase Event:**
- [ ] ✅ Название события: `Purchase` (не `Lead`)
- [ ] ✅ Параметр `content_type`: `product`
- [ ] ✅ Есть `contents` (массив товаров)
- [ ] ✅ Есть `value` и `currency`
- [ ] ✅ Advanced Matching: минимум 4 параметра (5 если авторизован)
- [ ] ✅ Event Source: `Web`
- [ ] ✅ В списке показывается сумма 💰

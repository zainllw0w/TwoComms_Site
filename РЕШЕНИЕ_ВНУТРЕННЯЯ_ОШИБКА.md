# 🔧 Решение: Internal Error (Внутренняя ошибка)

## 🚨 Что произошло?

Вы получили ошибку **"Internal Error"** - это общая ошибка сервера. Нужно выяснить причину.

---

**Вопросы для уточнения:**
- Открывали сайт в браузере? (какая страница?)
- Запускали команду через SSH?
- Настраивали cron через cPanel?
- Пытались запустить скрипт?

## 🛠️ Возможные причины и решения

### Причина 1: Ошибка в cPanel при добавлении cron

**Симптомы:** Ошибка при сохранении cron задачи в cPanel

**Решение:**
1. Проверьте команду - возможно слишком длинная
2. Упростите путь к логу: используйте `/tmp/nova_poshta_update.log`
3. Попробуйте добавить через SSH (см. ниже)

### Причина 2: Проблема с путями в cron команде

**Симптомы:** Cron не запускается или падает с ошибкой

**Решение:** Используйте упрощённую команду:

```bash
cd /home/qlknpodo/TWC/TwoComms_Site/twocomms && /home/qlknpodo/virtualenv/TWC/TwoComms_Site/twocomms/3.13/bin/python manage.py update_tracking_statuses >> /tmp/nova_poshta_update.log 2>&1
```

### Причина 3: Отсутствует директория для логов

**Решение:**

```bash
# Создать директорию
mkdir -p /home/qlknpodo/TWC/TwoComms_Site/twocomms/logs

# Проверить права
chmod 755 /home/qlknpodo/TWC/TwoComms_Site/twocomms/logs
```

### Причина 4: Ошибка в коде Python

**Решение:**
1. Проверьте логи (см. выше)
2. Запустите команду вручную для диагностики:

```bash
cd /home/qlknpodo/TWC/TwoComms_Site/twocomms
source /home/qlknpodo/virtualenv/TWC/TwoComms_Site/twocomms/3.13/bin/activate
python manage.py update_tracking_statuses
```

## ✅ Быстрое решение (через SSH)

Если не получается через cPanel, настройте через SSH:

```bash
sshpass -p '[REDACTED_SSH_PASSWORD]' ssh -o StrictHostKeyChecking=no qlknpodo@195.191.24.169 << 'ENDSSH'
cd /home/qlknpodo/TWC/TwoComms_Site/twocomms

# Создать директорию для логов если нет
mkdir -p logs

# Обновить cron задачу
(crontab -l 2>/dev/null | grep -v "update_tracking_statuses" | grep -v "Nova Poshta"; \
echo "# Nova Poshta: проверка статусов каждые 5 минут"; \
echo "*/5 * * * * cd /home/qlknpodo/TWC/TwoComms_Site/twocomms && /home/qlknpodo/virtualenv/TWC/TwoComms_Site/twocomms/3.13/bin/python manage.py update_tracking_statuses >> /tmp/nova_poshta_update.log 2>&1") | crontab -

# Проверить что добавилось
echo "=== Проверка cron задач ==="
crontab -l | grep update_tracking_statuses

echo ""
echo "✅ Готово! Cron настроен каждые 5 минут"
ENDSSH
```

## 💡 Упрощённая команда для cPanel

Если в cPanel не помещается длинная команда, используйте:

```
*/5 * * * * /home/qlknpodo/virtualenv/TWC/TwoComms_Site/twocomms/3.13/bin/python /home/qlknpodo/TWC/TwoComms_Site/twocomms/manage.py update_tracking_statuses >> /tmp/nova_poshta.log 2>&1
```

**Важно:** В cPanel может потребоваться указать полный путь без `cd`, так как некоторые cPanel не поддерживают `&&`.
